FUNDAMENTOS WEB + GIT
Descripción del proyecto

Este repositorio fue creado como parte de una actividad autodidacta para comprender los fundamentos del desarrollo web y el uso de Git en proyectos reales. Aquí se documentan conceptos básicos como Frontend, Backend, APIs REST, métodos HTTP, JSON y el flujo de trabajo con GitHub.

Frontend vs Backend

El Frontend es la parte visual de una aplicación web con la que interactúa el usuario. Incluye tecnologías como HTML, CSS y JavaScript.

El Backend es la parte lógica del sistema que procesa datos, maneja servidores y bases de datos. Se encarga de la seguridad, autenticación y funcionamiento interno de la aplicación.

¿Qué es una API REST?

Una API REST es una interfaz que permite que diferentes aplicaciones se comuniquen entre sí mediante solicitudes HTTP. Utiliza rutas o endpoints para enviar y recibir información.

Métodos HTTP

GET: Obtiene información del servidor.

POST: Envía nuevos datos al servidor.

PUT: Actualiza información existente.

DELETE: Elimina datos del servidor.

JSON

JSON (JavaScript Object Notation) es un formato ligero de texto utilizado para intercambiar datos entre aplicaciones. Es fácil de leer y muy usado en APIs.

Ejemplo básico:
{
"nombre": "Nayib",
"rol": "Estudiante"
}

¿Qué es Git y para qué sirve?

Git es un sistema de control de versiones que permite registrar cambios en un proyecto, trabajar en equipo y mantener un historial del código.

Conceptos básicos de Git

Repositorio: carpeta del proyecto donde se guardan archivos y cambios.
Commit: registro de cambios realizados.
Push: subir cambios al repositorio remoto.
Pull: descargar cambios del repositorio remoto.

¿Cómo funciona una API?

Una aplicación envía una solicitud HTTP a un endpoint. El servidor procesa la petición y responde con datos, normalmente en formato JSON.

¿Para qué sirve Git en un proyecto real?

Git permite:

Controlar versiones del código.

Trabajar colaborativamente.

Evitar pérdida de información.

Volver a versiones anteriores si ocurre un error.
